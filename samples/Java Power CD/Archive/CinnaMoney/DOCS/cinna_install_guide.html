Install distribution:

These instructions presume that you have already installed the Java Developer's Kit (CinnaMoney was built with FCS1.0 of the JDK, if you don't already have it, you should download it now.) in some directory that will be referred to through the rest of this document as 

Sun Solaris:
cd <java home directory>
uncompress -c cinnamoney-fcs1.tar.Z | tar xpfv -

(Warning, dos versions of pkunzip will truncate filenames rendering this package useless - you need an upgraded version of unzip which will support long filenames Winzip 6.0 if you don't already have it is a great package) 

Windows:

cd <java home directory>
pkunzip -d cinnamoney-fcs1.zip

Using the package:

A simple Test program is included in this distribution using String Objects just to help you to see if you installed the program properly: 

// TestCinnaMoney.java
// Test Program to test if CinnaMoney Installed


import java.util.*;
import java.lang.*;
import java.lang.*;
import CinnaMoney;

public class TestCinnaMoney {
	public static void main(String args[]) {
	String strn1 = new String("12000.00");
	CinnaMoney m1 = new CinnaMoney(strn1);
	String strn2 = new String("-1200.00");
	CinnaMoney m2 = new CinnaMoney(strn2);
	CinnaMoney m3;
	m3 = m1.add(m2);
	System.out.print(m1.toString());
	System.out.print(" + ");
	System.out.print(m2.toString());
	System.out.print(" = ");
	System.out.println(m3.toString());
	m3 = m1.subtract(m2);
	System.out.print(m1.toString());
	System.out.print(" - ");
	System.out.print(m2.toString());
	System.out.print(" = ");
	System.out.println(m3.toString());
	m3 = m1.divide(m2);
	System.out.print(m1.toString());
	System.out.print(" / ");
	System.out.print(m2.toString());
	System.out.print(" = ");
	System.out.println(m3.toString());
	m3 = m1.multiply(m2);
	System.out.print(m1.toString());
	System.out.print(" * ");
	System.out.print(m2.toString());
	System.out.print(" = ");
	System.out.println(m3.toString());
	System.out.println(m3.toString(2));
	}
}
To see if This compiles correctly do the following - it is presumed that your CLASSPATH contains a directory reference to "." in it's list of directories. If this is not the case, then the java runtime might have trouble finding this test program when you try to run it with the Java package. 
javac TestCinnaMoney.java
Java TestCinnaMoney

This should print:
