Java KV diagram and logic minimization demonstration

This page demonstrates KV diagrams and logic minimization. It is intended for our computer science undergraduate students.

Java Beta-1 version

The applet uses different colors to draw the KV loops and the corresponding PLA gates. They will only display correctly on a color display and with Netscape 2.0b (Java-enabled).

Due to bugs in Netscape 2.0b this page has redraw problems on Win'95. You may try to leave and re-enter the page if this happens.

The applet works fine with the SUN appletviewer.

Applet basics:

  • The applet at the bottom of this page demonstrates KV diagrams and logic minimization. For a detailed explanation of KV (Karnaugh Veitch) diagrams see any standard computer science or electrical engineering textbook.

  • The basic idea of a KV diagram is to represent a given boolean function as a kind of function table. At first, the function is represented using minterms. In order to minimize the function, one has to combine minterms to larger terms.

    The applet demonstrates this, because the current two-level logic realization of the function is plotted in a seperate window. You can watch how the function schematic gets simpler (and the logic realization gets cheaper) while you minimze the function.

  • The way to minimze a logic function using a KV diagram is to draw loops, in order to combine several minterms to larger terms. Larger terms are cheaper than minterms in the logic realization!

Applet usage:

  • The applet consists of the main window, and two seperate PLA windows. The main window contains a row of buttons on top, the KV diagram frame in the center, and a status line on the bottom.

    The two seperate PLA windows are used to display the schematic for the current function (that is, a two level AND-OR realization for the function), and the PLA table in the standard PLA (Espresso.5) format.

    The status line displays a kind of 'one line help messages' on what kind of mode the applet is in, and what you can (and should) do with the applet.

  • Click on the New: button to create a random new logic function with 2 to 6 inputs. The corresponding KV diagram without loops is drawn, and the PLA frame window displays the function realized with minterms.

    If you want to enter your own function, you may have to click a few times on the New: button, until the random function has the correct number of inputs. Afterwards use Edit Mode to enter the desired function.

  • Click on the Edit function button to enter Edit Mode. In this mode, the value of f[i] is toggled between 0 and 1 when you click on the KV diagram field i.

    This mode also allows you to enter a desired function bit by bit, if you want to use the KV diagram applet for actual logic minimization.

  • Click on the Add Loops button to enter Add Loop Mode. In this mode, you can start a new loop with index i, when you click on the KV diagram field i.

    To enlarge the loop, hold down the SHIFT key on your keyboard and click on the field you want to add to the current loop. The applet will automatically calculate all fields enclosed in the current loop (or warn you, if the clicked field cannot be added to the current loop).

    Loops are drawn in different colors (which repeat after 8 loops), in order to allow easy recognition of loops.

    Any time a loop is drawn, the PLA schematic and table are updated. Loops and the correspondings AND-gates are drawn in the same color.

  • If you want to undo a loop, or to delete all loops, click on the Delete Loop button to enter Delete Loop Mode.

    On each click, this will delete the last loop drawn. Each deleted loops is split into the corresponding minterms, and the PLA schematic and table are updated.

  • Click on the Highlight button to enter Highlight Mode. When you click on a field of the KV diagram in Highlight Mode, the active AND terms for this field (for all loops enclosing this field) are highlighted in the PLA schematic.

    Also, the numerical index (0 .. 63) of the KV diagram field in displayed in the Status Line on the bottom of the applet. This makes it easy to find a certain index field in the diagram, just by clicking a few times.

    (Note for Computer Science profs: The layout of the diagrams was deliberately modified somewhat from the standard layout. Using the applet for doing the homework will therefore still need some understanding...)
