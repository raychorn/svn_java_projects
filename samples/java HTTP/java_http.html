Java HTTP Classes

Retrieving Content

The class HTTPConnect (source) actually handles the mechanics of connecting to an HTTP server to retrieve a URL. It must be given an object implementing HTTPClient (source) when created. When run, HTTPConnect creates the connection, then calls functions from it's HTTPClient which provide necessary information: a host to connect to, the file on the host, the transaction method (GET, HEAD, or POST), and the content for POST requests. HTTPConnect parses the HTTP header, and then turns control over to the HTTPClient to actually handle the content. Finally, HTTPConnect closes the connection.

With these two classes, you can retrieve any sort of material from an HTTP server. The program HTMLEcho (source), for example, lets you specify a URL, and name/value pairs for POST requests, and returns HTML or Ascii text from that location. It implements HTTPClient, and starts a new HTTPConnect for each URL.
